# 東大蔵書チェッカー — 本のバーコードで書誌と東大OPACを引く

本の裏のバーコード（ISBN）をスマホのカメラで読み取り、**書名・著者・書影**を表示して、
**東大OPACのISBN検索リンク**を返すPWA（Webアプリ）です。

サーバー不要・ビルド不要・APIキー不要。`index.html` 一枚で完結します
（依存はバーコード読取の `html5-qrcode` のみ）。

## 仕組み（アーキテクチャ）

```
[スマホのカメラ]
   │  ① EAN-13バーコード → ISBN(978…) を抽出
   ▼
[index.html / PWA] ──② 書誌・書影──▶ openBD (api.openbd.jp)  ※ブラウザから直接（CORS可）
   │                              └ 予備: Google Books
   ▼
   ③ 書名・著者・出版社・書影を表示
   │
   ▼
[PWA] ＋ [東大OPACで開く] リンク（ISBN索引へのディープリンク）を表示
```

### なぜこの構成か
- **書誌・書影は openBD**：登録不要・CORS可でブラウザから直接呼べる。取得できないときは Google Books に自動フォールバック。
- **所蔵・貸出はOPACのリンクのみ（叩かない）**：東大OPACは自動アクセス（スクレイピング）を robots.txt で禁止し、
  過剰アクセスを能動的にブロックしているため、**コードで叩かず、ユーザーが自分のブラウザで開くリンク**を返す。
  これでリアルタイムの所蔵・貸出状況も“1タップ”で確認でき、規約・サーバー負荷の両面で安全。
- **中継サーバー不要**：使うAPIはどちらもブラウザから直接呼べるため、静的ホスティングだけで動く。

## データソースと役割
| ソース | 用途 | 備考 |
|---|---|---|
| openBD | 書名・著者・出版社・**書影** | 登録不要・高速・CORS可（書誌が無ければ Google Books に自動フォールバック） |
| Amazon 書影 | **書影**（openBDに書影が無いとき） | ISBN-10 を画像URLに用いる。画像が無い場合は自動でプレースホルダ表示 |
| 東大OPAC | 所蔵・貸出状況の確認 | リンク先として開く（自動アクセスは禁止） |

---

## セットアップ

`index.html` を静的ホスティング（Cloudflare Pages / Netlify / GitHub Pages 等）に置くだけです。
**カメラはHTTPSが必須**なので、ローカルの `file://` では動きません（`http://localhost` は可）。

APIキーや設定は不要です。

---

## 使い方
1. 「カメラで読み取る」→ 本の裏の**上のバーコード（978で始まる方）**を枠に合わせる
   （下の `192…` は価格用なので自動で無視されます）。ISBNを直接入力することもできます。
2. 書影・書名・著者・出版社が表示されます。
3. **[東大OPACで開く]** をタップして所蔵・貸出状況を確認します。

## 既知の制約・注意
- openBD / Google Books に書誌が無い本（ごく最近の本・一部の電子ブック等）は「書誌情報なし」と表示されますが、
  OPACリンクは ISBN から生成されるため引き続き利用できます。
- 所蔵・貸出の判定はアプリ側では行いません。実際の所蔵・在架状況は OPAC リンク先でご確認ください。

## 拡張アイデア
- スキャン履歴・「読みたい」リスト（PWAのIndexedDB）。
- OPAC以外の検索リンク（NDL サーチ、CiNii Books 等）の併設。
